C++ Writing Style

Correct structure is critical. Below is a comparison of valid vs invalid code structure.

✓ Correct Style

#include <iostream> #include <string> using namespace std; // This program prints out "Hello World!" int main() { string message = "Hello World!\n"; cout << message; return 0; }

✕ WRONG!

using namespace std; #include <iostream> #include <string> string M= "Hello World!\n"; int main() { // Does some stuff cout << M;; return (3); }

Include Statement Order

Include headers should be listed in the following order:

  1. C System Headers (e.g., <stdlib.h>)
  2. C++ Standard Library Headers (e.g., <iostream>)
  3. User-Defined Libraries' Headers (e.g., "foo/server/bar.h")

Naming Convention & Punctuation

Names can never start with a digit. Avoid reserved keywords.

PascalCase

For Classes & Functions.

LinkedList, BubbleSort()

snake_case

For Variables.

student_id, result

Punctuation Marks

  • Brackets {}: Open on same line, close under last line of scope.
  • Parentheses (): Space before ( in statements, no space in functions.
  • Commas ,: Always a blank space after each comma.
int GetLargerNumber(int num_one, int num_two) { if (num_one > num_two) { return num_one; } else { return num_two; } }

// Single-line: The compiler ignores text after // cout << "Codecademy rocks!"; /* Multi-line comments begin with /* and end with */ int multiply(int a, int b) { return a * b; }

std::tuple

Fixed-size collection of heterogeneous values. Unlike pairs, tuples can hold unlimited elements.

// 1. Creation std::tuple<int, string, double> student(1, "Alice", 3.8); // 2. Accessing int id = std::get<0>(student); // 3. Modifying std::get<2>(student) = 4.0;

Helper Functions

  • std::make_tuple: Create tuple without explicit types.
  • std::tie: Unpack tuple into variables.
  • std::tuple_cat: Concatenate tuples.

Arrays vs Vectors

std::vector (Dynamic)

Use when size is unknown.

vector<int> row(n); for(int i=0; i<n; i++) { cin >> row[i]; }

std::array (Fixed)

Better performance, known size.

array<int, 3> row; for(int i=0; i<3; i++) { cin >> row[i]; }

Memory: References & Pointers

References (&)

An alias to an existing variable. Must be initialized.

int exam_grade = 85; int& score = exam_grade; // score is alias for exam_grade

Warning: References cannot be reassigned!

Pointers (*)

Stores a memory address. Can be nullptr.

int* ptr = nullptr; string day = "Monday"; string* ptr = &day; // & = Address Of cout << *ptr; // * = Dereference (Get Value) -> "Monday"

Switch Statement

switch (grade) { case 9: cout << "Freshman\n"; break; case 10: cout << "Sophomore\n"; break; default: cout << "Invalid\n"; break; }

Functions

Structure

  • Declaration: Name, Return type, Params.
  • Definition: Body code.

Overloading

Same name, different parameters.

int add(int a, int b)
double add(double a, double b)

Interactive: Pass by Reference vs Value

If you pass by value, the swap happens only inside the function. If you pass by reference (&), the original variables change.

Command Line Arguments

int main(int argc, char* argv[])

argc: Argument Count. argv: Argument Vector (Array).

$ ./greeting "Code cademy"

Hash Maps (Super-Arrays)

Array looks up by Index. Hash Map looks up by Key (Identity).

// "Sum Value" -> "Count" unordered_map<int, int> seenSums; seenSums[5] = 1; if (seenSums.count(5)) { /* Found! */ }

Classes & Objects

OOP encapsulates data and functionality. Example: School class.

class School { public: string name; int age; void banner() { cout << name << " is " << age << " years old.\n"; } };

Multiple Objects

School harvard;
harvard.name = "Harvard";
harvard.age = 385;

School yale;
yale.name = "Yale";
yale.age = 319;

Encapsulation & Accessors

Access Specifiers

  • private: Default. Not accessible outside class.
  • public: Accessible everywhere.

Getters & Setters

Use public functions to access private variables.

class Clock { private: int time = 1200; public: // Accessor (Getter) int getTime() { return time; } // Mutator (Setter) void setTime(int new_time) { time = new_time; } };

Constructors & Destructors

Special methods called on object creation/destruction.

class House { private: string location; int rooms; public: // Constructor with Default Params House(string loc = "New York", int num = 5) { location = loc; rooms = num; } // Destructor ~House() { cout << "Moved away from " << location; } };

Member Initializer Lists

Required for const variables.

Book() : title("Diary"), pages(100) {}
